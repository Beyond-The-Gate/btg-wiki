# Systems

Game design translated into concrete, non-technical feature descriptions — the bridge between **[Game Design](../game-design/index.md)** and **[Technical](../technical/index.md)**.

Each system page should answer: *what does this do, from a player's perspective?*

## Example systems

- **Dungeons** — every player owns one; rooms, gates, trusted members.
- **Friends** — requests, friendships, live notifications.
- **Collections** — long-term progression by gathering resources, monsters, bosses, gates.
